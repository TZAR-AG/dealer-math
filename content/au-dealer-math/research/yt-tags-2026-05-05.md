## Methodology

Two-phase pull, both data-backed:

**Phase 1 — Competitor tag scrape** via Scrapling (curl-cffi fast fetcher) hitting 15 confirmed competitor video URLs in `<meta name="keywords">` extraction. Raw HTML preserved at `raw/*.html` for forensic re-extraction. JSONL of all extracted tags at `raw/competitor-tags.jsonl`.

**Phase 2 — YouTube autocomplete mining** via `suggestqueries.google.com/complete/search?client=youtube&ds=yt&q=…` (JSONP-wrapped JSON). 64 seed queries across two batches → 569 unique autocomplete tails. Raw at `raw/yt-autocomplete.json` + `raw/yt-autocomplete-batch2.json`. Combined synthesis pool at `raw/synthesis-pool.json`.

**No 3rd-party tag tools used** (rapidtags.io / keywordtool.io extractors 404'd on direct URL paths — auth-walled or moved). Direct `<meta name="keywords">` extraction worked for 8 of 15 videos (Cadogan top-10 + top-20 + EOFM, both Lopez videos, both F&I-coach videos). The remaining 7 returned YouTube's default-fallback tags (`video, sharing, camera phone, video phone, free, upload`) which are NOT real tag data — they're the platform's silent-fallback when no `<meta name="keywords">` is set on the watch HTML. We treat these as "no data" rather than real tags.

### Char-budget rule

Every V[N] tag string MUST be **480-498 YT-counted chars**. Run the helper before publish:

```python
def yt_count(tags):
    # Empirically calibrated 2026-05-05 against YT Studio's live counter
    # for V2 Shorts 1-5: actual count ≈ plain string length + ~20 chars.
    # The +20 is roughly 1 char/tag overhead (likely comma normalisation
    # or internal storage padding — NOT quote-wrapping as initially assumed).
    return len(', '.join(tags)) + len(tags)
```

**Target plain string length = 460-478** to land in the 480-498 actual YT range.

If under: add 1-2 more autocomplete-confirmed long-tails to slot 5-7 pool. If over: drop the weakest universal in slot 18-20 pool.

## Methodology notes — what worked, what didn't, gaps

### What WAS done

- Pulled `<meta name="keywords">` from 15 competitor videos via Scrapling — 8 returned real tags, 7 returned YT-default fallback
- Mined YT autocomplete for 64 seed queries → 569 unique tails
- Filtered noise (geo-irrelevant, competitor-brand, Google-misinterpreted seeds)
- Built char-budget-validated tag strings for V1 + V2 (490-493 of 500 YT-counted)
- Codified reusable slot formula for V3-V25

### Gaps + caveats

1. **7 of 15 videos returned YT-default fallback tags** (`video, sharing, camera phone, video phone, free, upload`). This is YouTube's silent-fallback when no `<meta name="keywords">` is set, NOT real tag data. We treat as no-data. Most-recent CarEdge / Lopez videos and many newer Cadogan videos have moved off the meta-keyword path — possibly using only YouTube's internal keyword-graph entity now, which isn't extractable from public HTML. Mitigation: 8-video sample is still enough to establish high-frequency multi-source patterns; the autocomplete mining (Phase 2) compensates with broader user-search-side data.

2. **3rd-party tag-extraction sites (rapidtags.io, keywordtool.io) 404'd or required login.** Skipped without burning further retries (per `feedback_re_read_file_before_iterating_2026-05-04.md` — don't iterate blind). If we hit a wall on V12-review where we need richer competitor tag data, the path is to re-run `generator/scrapling/yt-tag-pull.py` against newer competitor URLs OR to widen the autocomplete-seed list with V1-V11 actual Studio search-query data. **No paid SaaS** (TubeBuddy / vidIQ / etc) — channel runs free per `feedback_no_new_saas_until_revenue.md`.

3. **Aussie Finance With Luke channel** — couldn't surface ranked video URLs through search (~thin web presence indexed). His tag list would be the closest direct competitor; deferred to V12 review when AUDM has its own data to anchor against.

4. **Autocomplete is search-side, not algorithm-side.** It tells us what users TYPE; it doesn't tell us what tags YouTube indexes those queries to. The real validation comes from V1-V12 Studio data: which tags drive the most browse-feed impressions vs search-feed impressions. **Re-score this entire formula at V12** with that empirical data.

5. **`f&i office` tag uses `&` symbol.** YouTube indexes `&` and `and` separately. A safer hedge would be to include both `f&i office` AND `f and i office` — but we'd burn 2 slots for one concept. Decision: lead with `f&i office` (industry-standard term, matches CarEdge / fi_top1_routine pool) + slot 3 `finance and insurance officer` covers the spelled-out variant via autocomplete-confirmed term. If V2 underperforms on F&I-related search impressions in first 7 days, swap slot 3 → `f and i office`.

6. **Slot 18 `new car buying tips` reads US-leaning** (Australians say "buying a car" more than "new car buying tips"). Kept because it's the single highest-multi-source competitor tag (cadogan_top10 + cadogan_top20). Re-evaluate at V12: if AU-only impression share is <60% on V[N]s, drop this slot in favor of `australian car buying`.
